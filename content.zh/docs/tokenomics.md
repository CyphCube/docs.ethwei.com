# 代币经济

Ethwei 的代币模型设计得可预测且无聊。没有复杂的解锁悬崖，没有由治理控制的通胀，没有意外。

## 代币：ETE

| 属性 | 数值 |
|---|---|
| 代号 | **ETE** |
| 最小单位 | `WEI` |
| 精度 | 6（1 ETE = 1,000,000 WEI） |
| 创世供应量 | 7,000,000,000 ETE（70 亿） |
| 最大供应量 | 10,000,000,000 ETE（100 亿） |
| 年增发量 | 100,000,000 ETE / 年（1 亿） |
| 增发周期 | 30 年 |
| 每区块增发 | 约 15.84 ETE / 区块 |
| 社区税 | **0%** |

## 供应计划

该链从创世开始，每年固定增发 **100,000,000 ETE**，持续 30 年。
这不是基于通胀率的，而是一个固定的绝对数量，与总供应量无关。

```
第 0 年    7,000,000,000 ETE（创世）
第 1 年    7,100,000,000 ETE
第 5 年    7,500,000,000 ETE
第 10 年   8,000,000,000 ETE
第 20 年   9,000,000,000 ETE
第 30 年  10,000,000,000 ETE（达到最大供应量）
```

30 年后，增发停止。此后该链仅依靠交易手续费继续运行。

## 区块奖励

在约 6.3 秒的出块时间下（CometBFT 默认值）：

```
每年区块数 ≈ 5,000,000
每区块增发 ≈ 100,000,000 / 5,000,000 = 20 ETE
```

> [!NOTE]
> 规范中写的是约 15.84 ETE/区块，对应约 6,313,131 个区块/年
> （更接近约 5 秒的平均出块时间）。你实际获得的区块奖励取决于
> 链上实测的出块时间。

## 分配

所有区块奖励都按质押比例分配给**验证人及其委托人**。
没有社区池税、基金会抽成或协议手续费。

```
区块奖励
  └─ 100% 给验证人集合
       ├─ 验证人佣金（由每个验证人各自设定）
       └─ 委托人奖励（按质押比例）
```

## 关于计价单位的说明

在链上，所有金额都以 `WEI`（最小单位）计。当你看到一笔金额为
`1000000 WEI` 的交易时，那就是 `1 ETE`。钱包和区块浏览器会以 `ETE` 单位显示，方便阅读。
